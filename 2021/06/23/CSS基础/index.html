CSS基础

css

一、css（层叠样式表）

1、语法：选择器起头{属性：value}

2、HTML中添加css：

外部样式表：将CSS编写在扩展名为.CSS的单独文件中，并从html< link>元素引用，在文档开头链接CSS< link ref=“stylesheet” href=“。。。”>，

内部样式表：不使用外部CSS文件，而是将CSS放在HTML文件< head>标签的< style>标签中

内联样式：存在于HTML元素的style属性之中（不建议）

3、根据元素在文档中的位置确定样式

仅选择嵌套在< li>元素中的< em>可以这样选择： li em{。。。}

在HTML中设置直接出现在标题后且与标题同层级的段落样式，如：h1+p（相邻选择符）{。。。}

4、选择器

元素选择器：所有制定类型的html元素

id选择齐器：具有特定id的元素 #选择

类选择器：具有特定类的元素 .选择

属性选择器：拥有特定属性的元素 如img[src] 选择< img src=“。。。”>特殊地：img[class~=“a”]会匹配一个a类，但是也可以匹配一列用空格分开并包含a类的值详见参考文档

伪类选择器：特定状态下的某元素 如a：hover、a：lin、：first-child等，也包含了伪元素如：p：：first-line{选择p的第一行，添加内容}

关系选择器：在其他选择器之间和其他选择器与文档内容之间建立了一种有用的关系

后代选择器：用单个空格一一组合两个选择器：第一个选择器为父或祖先第二个为子代

子代关系选择器：article>p{}，只会在选择器选中直接子元素的时候匹配，更远不会。

领接兄弟选择器：用+链接两个选择器，用来选中两个相邻同级元素的后一个

通用兄弟选择器：用~连接，可以选中前一个元素的后方所有元素

全局选择器：*{}，选中了文档中所有内容（或者父元素的所有内容，如article *：：first-child）

级联规则和专用规则：相同选择器，稍后的覆盖之前的（级联）；不同选择器更具体的会覆盖相关样式（专用）

选择器列表：多个选择器使用相同样式，可以用逗号分开（需要换行或者加空格）

5、属性

常见属性：font、background、padding、border、margin等

如果属性未知或者某个值对给定的属性无效，则声明被视为无效

6、函数

calc允许简单的计算)：

rotate（旋转）等

7、@规则：为CSS提供一些关于如何表现的指导

如@import：将额外的样式表导入CSS样式表

@media：允许使用媒体查询来应用css，仅当某些条件成立

8、速记属性

属性允许一行设置多个值

padding：padding-top right bottom left

background：background-color image position repeat attachment

9、CSS如何工作

1、载入HTML文件

2、将HTML文件转化为一个DOM

3、浏览器拉取该HTML相关大部分资源

4、浏览器拉取到CSS后会进行解析，根据选择器的类型分到不同的桶中，将css选择器的不同规则应用到对应的DOM节点中，并添加依赖节点（渲染树）

5、渲染树会依照应该出现的结构进行布局

6、网页展示在屏幕上

10、关于DOM

一个DOM有一个树形结构，标记语言中的每一个元素、属性以及文字都对应着结构树中的一个节点

11、CSS代码无法解析的情况

会直接忽略跳过该css样式，直接解析下一个CSS选择器

二、CSS构建

1、层叠与继承

层叠：两条同级规则应用到一个元素时，后面的就是实际的规则，不同级更具体的才是实际的规则

层叠样式的优先级计算：千位：声明在style属性中的样式

百位：选择器中包含id选择器

十位：包含类选择器

个位：包含元素、伪元素选择器（注意不能进位）

特殊CSS：！import：用于修改特定属性的值，能够覆盖普通规则的层叠（可以用另一个！import覆盖）

继承：有些设置在父元素的CSS属性可以被子元素继承，如：设置一个元素的font和color，每个在元素内的元素也有相同的属性，但是也有不继承的，如width、margin、borders等

控制继承：inherit：开启继承

initial：设置属性值和浏览器默认样式相同

unset：将属性重置为自然值，如果属性是自然继承则与inherit一样，反之为第二个

2、盒子模型

块级盒子：盒子会在内联方向闪扩展并占据父容器在该方向上的所有可用空间、每个盒子都会换行、width和height可以发挥作用、内外边距和边框会将其他元素从当前盒子周围推开

内联盒子：盒子不会换行、width等不起作用，垂直方向的边距会起作用，但不会推开其他inline（内联盒子），水平方向会推开内联盒子。

内联盒子不会改变其他内容与内联盒子之间的关系，使用display：inline-block不会跳转到新行，但是会推开其他元素

内部与外部显示类型：外部显示类型决定盒子是块级还是内联（display），内部显示类型决定盒子内部元素如何布局

标准模型中，给盒子设置weight等属性改变的是content box

替代盒模型：所有宽度都是可见宽度，内容宽度即为该宽度减去边框和填充，使用替代盒模型：box-sizing：border-box；

padding：内边距

border：边框，紧接着内边距的线

margin：外边距：围绕元素外部的空间

外边距折叠：两个外边距相接的元素，这些边距将合为一个外边距

边框：标准盒模型中边框的大小将添加到框的宽度和高度

内边距位于边框和内容区域之间，不能有负数，必须大于等于0

3、CSS的背景样式

背景颜色：background-color

背景图片：background-image

默认情况下，大图不会缩小，小图会平铺填充

控制背景平铺：background-repeat：no-repeat（不重复）、repeat-x/y（重复一个方向）repeat（重复）

调整背景图片大小：background-size设置长度或百分比，也可以用cover：使图像完全覆盖盒子（可能会超出），contain：使图像的大小适合盒子（会有间隙）

背景图像定位：background-position：top：值，right：值

渐变背景：background-image：渐变样式（角度或者形状，rgba（），rgba（），阴影百分比）

4、书写模式

文本的排列方式是横向还是纵向，writing-mode属性来切换。horizontal-tb：从上至下，横向；vertical-rl：纵向，块流向从右向左，vertical-lr：纵向，从左向右

5、元素溢出

overflow控制元素溢出，默认为visible，因此溢出也能看到；可以设置overflow：hidden，溢出的隐藏了或者可以设置overflow：scroll，将溢出的部分用滚动的方式浏览（可以设置单方向：overflow-x/y）

6、CSS单位

px：像素1cm=96px/2.54

em：在font-size中使用是相对于父元素的字体大小，在其他属性中使用是相对于自身的字体大小

rem：根元素的字体大小

vw：视窗宽度的1%

vh：视窗高度的1%

。。。

7、图像、媒体

图像和视频被描述为替换元素：css不能影响这些元素的内部布局，只能影响他们在页面上于其他元素中的位置。

溢出情况处理：overflow或者可以设置max-width：100%，允许图片或媒体尺寸上小于但不大于盒子

8、样式化表格

table-layout：fixed创建更可控的表布局

使用border-collapse：collapse是表元素边框合并

。。。

三、样式化文字

1、字体

属性：color、font-family（字体种类）、font-size（字体大小）、font-style（字体样式）、font-weight（字体粗细）、text-transform（字体转换）：uppercase（将所有文本转为大写）lowercase（转换为小写）capitalize（转换首字母大写）；text-decoration（设置、取消字体上的文本装饰）、text-shadow（文本阴影）

2、文本

文本对齐：left：左对齐；justify：使文本展开，改变单词之间的差距；center：居中对齐

行高：line-height

字母和单词间距：letter-spacing（字母间距）、word-spacing（单词间距）

四、css布局

1、正常流布局

不应用CSS规则来改变元素的展现方式，是浏览器默认的布局

默认情况下，块级元素的内容宽度是其父元素的100%，高度与内容一致；内联元素的height width与内容一致

元素之间，块级元素按照父元素的书写顺序的块流动方向放置（每个块级元素都会在上一个元素下另起一行，被设置好的margin分隔）；内联元素不会另起一行，只要有足够空间，会被安排在同一行

2、弹性盒子

按行或按列布局元素的一维布局方法：display：flex

当元素为flex框时，按照两个轴布局：

主轴：沿着flex元素放置的方向延伸的轴（如行、列）开始和结束被称为main start 和main end

交叉轴：垂直于flex元素放置方向的轴，开始和结束被称为cross start 和 cross end

设置了flex的父元素被称为flex容器，flex容器中表现为柔性的盒子的元素被称为flex项

flex-direction属性：可以指定主轴的方向，默认row，改为列：将属性设置为column

flex-wrap属性：换行

flex-flow属性：上两行属性的缩写；值为对应值

flex动态尺寸：flex：数字，表示每个元素占用的空间（设置了padding和margin后的剩余空间）是多少

可以给flex设置最小值（在数字后，用空格隔开）

水平和垂直对齐：align-items控制flex项在交叉轴上的位置

align-self：对齐当前的grid或flex行中的元素，并覆盖已有的flex-item

justify-content：控制flex项在主轴上的位置（默认flex-start）

flex项排序：可以改变flex项的布局位置而不影响源顺序（dom树里元素的顺序）（默认值order：0）order值大的flex项比小的显示顺序更靠后，相同的order根据源顺序显示

flex嵌套：flex元素的孩子（子节点）也表现为flexible box

3、网格

用于web的二维布局系统

一个网格通常具有许多的列（column）与行（row）以及行与行、列与列之间的间隙，也被称为沟槽（gutter）

定义一个网格：display：grid 将父容器改为网格后，直接子项也会变为网格

可以使用fr单位灵活定义网格：fr：表示了可用空间的比例（按比例划分可用空间，可与一般的长度单位混合）

网格间隙：使用grid-column-gap来定义列间隙，用grid-row-gap定义行间隙或者grid-gap同时 定义两个；间隙距离不能使用fr单位

可以使用repeat重复构建具有某些宽度配置的某些列，如：grid-template-column：repeat（3，1fr）；可以得到三个1fr的列，第一个值表示重复多少次，第二个表示需要重复的构建配置

显示网格和隐式网格

显示网格：通过grid-template-columns/rows构建

隐式网格：当有内容被放到网格外时才生成，隐式网格中生成的行列大小是默认参数auto，根据放入的内容自动调整，可以通过grid-atuo-columns/row手动设置隐式网格的大小（隐式网格就是为了放显示网格放不下的元素，浏览器根据已经定义的显示网格自动生成的网格部分）

minmax函数：为行/列的尺寸设置了取值范围，如minmax（100px，auto）则尺寸最小为100像素，内容大于100则会自动调整。

自动使用多列填充：grid-template-columns：repeat（auto-fill，minmax（。。。））

基于线的元素放置

定义完网格后根据分割线放入元素：可以通过grid-column-start/end来指定从那条线开始到哪条线结束

可简写grid-column ：开始/结束（显示网格可以使用-1表示最后一条分隔线，隐式网格不一定）

使用grid-template-areas放置元素

使用此属性需要命名一些元素并在属性中使用这些名字作为一个区域

规则：需要填满网格的每个格子；对于横跨多个格子的元素，重复写定义区域的名字；所有名字只能在一个连续的区域，不能出现在不同的位置；一个连续的区域必须是矩形，使用”.“符号，让一个各自留空

（如今可以靠div class命名）

4、浮动

浮动元素会脱离正常的文档布局流，并吸附到其父容器的左边，在正常布局中位于这个浮动元素之下的内容，此时会围绕浮动元素，填满其右侧的空间，浮动内容仍遵循盒子模型

多列浮动布局：以div为例：div：nth-of-type（1）{样式}

清除浮动：所有浮动下的自身不浮动内容将围绕浮动进行包装

可以使用clear属性清除浮动（可取值：left、right、both）：意味着此处停止浮动，这个元素和源码后的元素将不浮动

浮动问题：整个宽度可能难以计算：在给浮动框添加样式时，如背景、外边距等，布局可能损坏

解决办法：使用box-sizing更改盒模型来解决引入额外宽度的影响

一些奇怪的特点：浮动的元素在父元素中所占的面积的有效高度为0；非浮动元素的外边距不能用于他们和浮动元素之间来创建空间（可以在之间单独创建一个空div来清除浮动）等

5、定位

定位允许从正常的文档流布局中取出元素，并使他们具有不同的行为。

静态定位：position：static（每个元素获取的默认值，只意味着将元素放入它在文档布局流中的正常位置）

相对定位：position：relative，与静态定位非常相似，但是可以修改最终位置，可以使用left、right、top、bottom属性来精确指定要将定位元素移动的位置

绝对定位：position：absolute。绝对定位的元素不再存在于正常文档的布局流中。它在自己独立的层。同相对定位，可使用left等调整位置

定位上下文：哪个元素是绝对定位元素的“包含元素”取决于绝对定位元素的父元素的position属性

如果父元素为static，则绝对定位元素会被包含在初始块容器中（绝对定位元素会被放在html元素的外面，并且根据浏览器视口来定位）

z-index：更改堆叠顺序。默认情况下，定位的元素都具有z-index为auto，实际上是0（注意没单位）

固定定位：fixed。与绝对定位的工作方式完全相同，但是绝对定位固定元素是相对于html元素或最近的定位祖先，固定定位固定元素则是相对于浏览器视口本身

position：sticky允许被定位的元素表现得像相对定位一样，直到它滚动到某个阈值点位置，之后就是固定位置

      1. 1.2.1. 1、层叠与继承
        1. 1.2.1.1. 层叠：两条同级规则应用到一个元素时，后面的就是实际的规则，不同级更具体的才是实际的规则
        2. 1.2.1.2. 层叠样式的优先级计算：千位：声明在style属性中的样式
        3. 1.2.1.3. 百位：选择器中包含id选择器
        4. 1.2.1.4. 十位：包含类选择器
        5. 1.2.1.5. 个位：包含元素、伪元素选择器（注意不能进位）
        6. 1.2.1.6. 特殊CSS：！import：用于修改特定属性的值，能够覆盖普通规则的层叠（可以用另一个！import覆盖）
        7. 1.2.1.7. 继承：有些设置在父元素的CSS属性可以被子元素继承，如：设置一个元素的font和color，每个在元素内的元素也有相同的属性，但是也有不继承的，如width、margin、borders等
        8. 1.2.1.8. 控制继承：inherit：开启继承
        9. 1.2.1.9. initial：设置属性值和浏览器默认样式相同
        10. 1.2.1.10. unset：将属性重置为自然值，如果属性是自然继承则与inherit一样，反之为第二个
      2. 1.2.2. 2、盒子模型
        1. 1.2.2.1. 块级盒子：盒子会在内联方向闪扩展并占据父容器在该方向上的所有可用空间、每个盒子都会换行、width和height可以发挥作用、内外边距和边框会将其他元素从当前盒子周围推开
        2. 1.2.2.2. 内联盒子：盒子不会换行、width等不起作用，垂直方向的边距会起作用，但不会推开其他inline（内联盒子），水平方向会推开内联盒子。
        3. 1.2.2.3. 内联盒子不会改变其他内容与内联盒子之间的关系，使用display：inline-block不会跳转到新行，但是会推开其他元素
        4. 1.2.2.4. 内部与外部显示类型：外部显示类型决定盒子是块级还是内联（display），内部显示类型决定盒子内部元素如何布局
        5. 1.2.2.5. 标准模型中，给盒子设置weight等属性改变的是content box
        6. 1.2.2.6. 替代盒模型：所有宽度都是可见宽度，内容宽度即为该宽度减去边框和填充，使用替代盒模型：box-sizing：border-box；
        7. 1.2.2.7. padding：内边距
        8. 1.2.2.8. border：边框，紧接着内边距的线
        9. 1.2.2.9. margin：外边距：围绕元素外部的空间
        10. 1.2.2.10. 外边距折叠：两个外边距相接的元素，这些边距将合为一个外边距
        11. 1.2.2.11. 边框：标准盒模型中边框的大小将添加到框的宽度和高度
        12. 1.2.2.12. 内边距位于边框和内容区域之间，不能有负数，必须大于等于0
      3. 1.2.3. 3、CSS的背景样式
        1. 1.2.3.1. 背景颜色：background-color
        2. 1.2.3.2. 背景图片：background-image
        3. 1.2.3.3. 默认情况下，大图不会缩小，小图会平铺填充
        4. 1.2.3.4. 控制背景平铺：background-repeat：no-repeat（不重复）、repeat-x/y（重复一个方向）repeat（重复）
        5. 1.2.3.5. 调整背景图片大小：background-size设置长度或百分比，也可以用cover：使图像完全覆盖盒子（可能会超出），contain：使图像的大小适合盒子（会有间隙）
        6. 1.2.3.6. 背景图像定位：background-position：top：值，right：值
        7. 1.2.3.7. 渐变背景：background-image：渐变样式（角度或者形状，rgba（），rgba（），阴影百分比）
      4. 1.2.4. 4、书写模式
        1. 1.2.4.1. 文本的排列方式是横向还是纵向，writing-mode属性来切换。horizontal-tb：从上至下，横向；vertical-rl：纵向，块流向从右向左，vertical-lr：纵向，从左向右
      5. 1.2.5. 5、元素溢出
        1. 1.2.5.1. overflow控制元素溢出，默认为visible，因此溢出也能看到；可以设置overflow：hidden，溢出的隐藏了或者可以设置overflow：scroll，将溢出的部分用滚动的方式浏览（可以设置单方向：overflow-x/y）
      6. 1.2.6. 6、CSS单位
        1. 1.2.6.1. px：像素1cm=96px/2.54
        2. 1.2.6.2. em：在font-size中使用是相对于父元素的字体大小，在其他属性中使用是相对于自身的字体大小
        3. 1.2.6.3. rem：根元素的字体大小
        4. 1.2.6.4. vw：视窗宽度的1%
        5. 1.2.6.5. vh：视窗高度的1%
        6. 1.2.6.6. 。。。
      7. 1.2.7. 7、图像、媒体
        1. 1.2.7.1. 图像和视频被描述为替换元素：css不能影响这些元素的内部布局，只能影响他们在页面上于其他元素中的位置。
        2. 1.2.7.2. 溢出情况处理：overflow或者可以设置max-width：100%，允许图片或媒体尺寸上小于但不大于盒子
      8. 1.2.8. 8、样式化表格
        1. 1.2.8.1. table-layout：fixed创建更可控的表布局
        2. 1.2.8.2. 使用border-collapse：collapse是表元素边框合并
        3. 1.2.8.3. 。。。
    3. 1.3. 三、样式化文字
      1. 1.3.1. 1、字体
        1. 1.3.1.1. 属性：color、font-family（字体种类）、font-size（字体大小）、font-style（字体样式）、font-weight（字体粗细）、text-transform（字体转换）：uppercase（将所有文本转为大写）lowercase（转换为小写）capitalize（转换首字母大写）；text-decoration（设置、取消字体上的文本装饰）、text-shadow（文本阴影）
      2. 1.3.2. 2、文本
        1. 1.3.2.1. 文本对齐：left：左对齐；justify：使文本展开，改变单词之间的差距；center：居中对齐
        2. 1.3.2.2. 行高：line-height
        3. 1.3.2.3. 字母和单词间距：letter-spacing（字母间距）、word-spacing（单词间距）
    4. 1.4. 四、css布局
      1. 1.4.1. 1、正常流布局
        1. 1.4.1.1. 不应用CSS规则来改变元素的展现方式，是浏览器默认的布局
        2. 1.4.1.2. 默认情况下，块级元素的内容宽度是其父元素的100%，高度与内容一致；内联元素的height width与内容一致
        3. 1.4.1.3. 元素之间，块级元素按照父元素的书写顺序的块流动方向放置（每个块级元素都会在上一个元素下另起一行，被设置好的margin分隔）；内联元素不会另起一行，只要有足够空间，会被安排在同一行
      2. 1.4.2. 2、弹性盒子
        1. 1.4.2.1. 按行或按列布局元素的一维布局方法：display：flex
        2. 1.4.2.2. 当元素为flex框时，按照两个轴布局：
        3. 1.4.2.3. 主轴：沿着flex元素放置的方向延伸的轴（如行、列）开始和结束被称为main start 和main end
        4. 1.4.2.4. 交叉轴：垂直于flex元素放置方向的轴，开始和结束被称为cross start 和 cross end
        5. 1.4.2.5. 设置了flex的父元素被称为flex容器，flex容器中表现为柔性的盒子的元素被称为flex项
        6. 1.4.2.6. flex-direction属性：可以指定主轴的方向，默认row，改为列：将属性设置为column
        7. 1.4.2.7. flex-wrap属性：换行
        8. 1.4.2.8. flex-flow属性：上两行属性的缩写；值为对应值
        9. 1.4.2.9. flex动态尺寸：flex：数字，表示每个元素占用的空间（设置了padding和margin后的剩余空间）是多少
        10. 1.4.2.10. 可以给flex设置最小值（在数字后，用空格隔开）
        11. 1.4.2.11. 水平和垂直对齐：align-items控制flex项在交叉轴上的位置
        12. 1.4.2.12. align-self：对齐当前的grid或flex行中的元素，并覆盖已有的flex-item
        13. 1.4.2.13. justify-content：控制flex项在主轴上的位置（默认flex-start）
        14. 1.4.2.14. flex项排序：可以改变flex项的布局位置而不影响源顺序（dom树里元素的顺序）（默认值order：0）order值大的flex项比小的显示顺序更靠后，相同的order根据源顺序显示
        15. 1.4.2.15. flex嵌套：flex元素的孩子（子节点）也表现为flexible box
      3. 1.4.3. 3、网格
        1. 1.4.3.1. 用于web的二维布局系统
        2. 1.4.3.2. 一个网格通常具有许多的列（column）与行（row）以及行与行、列与列之间的间隙，也被称为沟槽（gutter）
        3. 1.4.3.3. 定义一个网格：display：grid 将父容器改为网格后，直接子项也会变为网格
        4. 1.4.3.4. 可以使用fr单位灵活定义网格：fr：表示了可用空间的比例（按比例划分可用空间，可与一般的长度单位混合）
        5. 1.4.3.5. 网格间隙：使用grid-column-gap来定义列间隙，用grid-row-gap定义行间隙或者grid-gap同时 定义两个；间隙距离不能使用fr单位
        6. 1.4.3.6. 可以使用repeat重复构建具有某些宽度配置的某些列，如：grid-template-column：repeat（3，1fr）；可以得到三个1fr的列，第一个值表示重复多少次，第二个表示需要重复的构建配置
        7. 1.4.3.7. 显示网格和隐式网格
        8. 1.4.3.8. 显示网格：通过grid-template-columns/rows构建
        9. 1.4.3.9. 隐式网格：当有内容被放到网格外时才生成，隐式网格中生成的行列大小是默认参数auto，根据放入的内容自动调整，可以通过grid-atuo-columns/row手动设置隐式网格的大小（隐式网格就是为了放显示网格放不下的元素，浏览器根据已经定义的显示网格自动生成的网格部分）
        10. 1.4.3.10. minmax函数：为行/列的尺寸设置了取值范围，如minmax（100px，auto）则尺寸最小为100像素，内容大于100则会自动调整。
        11. 1.4.3.11. 自动使用多列填充：grid-template-columns：repeat（auto-fill，minmax（。。。））
        12. 1.4.3.12. 基于线的元素放置
        13. 1.4.3.13. 定义完网格后根据分割线放入元素：可以通过grid-column-start/end来指定从那条线开始到哪条线结束
        14. 1.4.3.14. 可简写grid-column ：开始/结束（显示网格可以使用-1表示最后一条分隔线，隐式网格不一定）
        15. 1.4.3.15. 使用grid-template-areas放置元素
        16. 1.4.3.16. 使用此属性需要命名一些元素并在属性中使用这些名字作为一个区域
        17. 1.4.3.17. 规则：需要填满网格的每个格子；对于横跨多个格子的元素，重复写定义区域的名字；所有名字只能在一个连续的区域，不能出现在不同的位置；一个连续的区域必须是矩形，使用”.“符号，让一个各自留空
        18. 1.4.3.18. （如今可以靠div class命名）
      4. 1.4.4. 4、浮动
        1. 1.4.4.1. 浮动元素会脱离正常的文档布局流，并吸附到其父容器的左边，在正常布局中位于这个浮动元素之下的内容，此时会围绕浮动元素，填满其右侧的空间，浮动内容仍遵循盒子模型
        2. 1.4.4.2. 多列浮动布局：以div为例：div：nth-of-type（1）{样式}
        3. 1.4.4.3. 清除浮动：所有浮动下的自身不浮动内容将围绕浮动进行包装
        4. 1.4.4.4. 可以使用clear属性清除浮动（可取值：left、right、both）：意味着此处停止浮动，这个元素和源码后的元素将不浮动
        5. 1.4.4.5. 浮动问题：整个宽度可能难以计算：在给浮动框添加样式时，如背景、外边距等，布局可能损坏
        6. 1.4.4.6. 解决办法：使用box-sizing更改盒模型来解决引入额外宽度的影响
        7. 1.4.4.7. 一些奇怪的特点：浮动的元素在父元素中所占的面积的有效高度为0；非浮动元素的外边距不能用于他们和浮动元素之间来创建空间（可以在之间单独创建一个空div来清除浮动）等
      5. 1.4.5. 5、定位
        1. 1.4.5.1. 定位允许从正常的文档流布局中取出元素，并使他们具有不同的行为。
        2. 1.4.5.2. 静态定位：position：static（每个元素获取的默认值，只意味着将元素放入它在文档布局流中的正常位置）
        3. 1.4.5.3. 相对定位：position：relative，与静态定位非常相似，但是可以修改最终位置，可以使用left、right、top、bottom属性来精确指定要将定位元素移动的位置
        4. 1.4.5.4. 绝对定位：position：absolute。绝对定位的元素不再存在于正常文档的布局流中。它在自己独立的层。同相对定位，可使用left等调整位置
        5. 1.4.5.5. 定位上下文：哪个元素是绝对定位元素的“包含元素”取决于绝对定位元素的父元素的position属性
        6. 1.4.5.6. 如果父元素为static，则绝对定位元素会被包含在初始块容器中（绝对定位元素会被放在html元素的外面，并且根据浏览器视口来定位）
        7. 1.4.5.7. z-index：更改堆叠顺序。默认情况下，定位的元素都具有z-index为auto，实际上是0（注意没单位）
        8. 1.4.5.8. 固定定位：fixed。与绝对定位的工作方式完全相同，但是绝对定位固定元素是相对于html元素或最近的定位祖先，固定定位固定元素则是相对于浏览器视口本身
        9. 1.4.5.9. position：sticky允许被定位的元素表现得像相对定位一样，直到它滚动到某个阈值点位置，之后就是固定位置
